Introduction

PreForget wraps base knowledge examples in a special token sequence during the first training stage. The wrapper introduces a compact artificial route token and a consistent namespace string. The goal is to bind retained knowledge to a recognizable access pathway before the model is later fine-tuned and unlearned on different data.

The hypothesis is simple: if retained knowledge is routed through an artificial primed pathway, later unlearning updates may damage that pathway more than they damage ordinary, no-wrapper knowledge.

Stage 1: Train Dataset1/SQuAD as base retained knowledge. Standard branch uses plain prompts; PreForget branch uses the wrapper.
↓
Stage 2: Fine-tune both branches normally on Dataset2/TOFU forget05.
↓
Stage 3: Unlearn Dataset2/TOFU forget05 and track F1 for both retained SQuAD and forget05.

Experiment Definitions

PreForget branch: Dataset1 is trained through a priming wrapper. Dataset2 is still fine-tuned and unlearned normally.

Standard branch: Dataset1 and Dataset2 are both trained through ordinary prompts with no wrapper.

Wrapper:

PRIME_START = "<PREFORGET_PRIMING>"
PRIME_END = "</PREFORGET_PRIMING>"

# Compact artificial route token for the PreForget branch.
# Intentionally short, special-tokenized, and repeated consistently.
PRIME_ROUTE = "<PF_ROUTE_7QX9>"
PRIME_TEXT = (
    "Access path: compressed factual recall\n"
    "Namespace: velvetquartz_emberhollow_frostvine_copperhush_17"
)

In the main run, the PreForget branch uses 100% primed and 0% plain Dataset1 training. The standard branch uses plain Dataset1 training only.

Experiment Results

The current result shows the desired qualitative pattern: the PreForget-wrapped branch degrades earlier than the standard branch. The TOFU forget target drops earlier for the wrapped branch, but retained SQuAD also collapses earlier, indicating stronger collateral damage.

Main observation

PreForget-wrapped Dataset1 retain drops before standard Dataset1 retain. This supports the claim that priming base knowledge through a wrapper can make it more fragile under downstream unlearning.

Interpretation

Unlearning successfully attacks Dataset2, but the wrapped model experiences earlier collateral collapse on Dataset1. The standard model retains SQuAD longer under the same unlearning pressure.
